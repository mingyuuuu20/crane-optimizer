# 실제 한국 사례 검증 계획 — 마포구 공덕동 일대

## 검증 목적
우리 시스템이 공덕동 256-42 가상 시뮬레이션을 넘어, **실제 시공 사례에 대해서도 합리적 추천** 을 내는지 정량 검증한다.

## 검증 데이터 소스

### 1차 공식 데이터 (정량)
- **공공데이터포털 — 서울특별시 마포구 민간 건축공사 착공신고 현황** 
  - URL: https://www.data.go.kr/data/15108020/fileData.do
  - CSV 다운로드 → 대지면적·건축면적·층수·주용도·도시형생활주택 여부
- **공공데이터포털 — 서울특별시 마포구 아파트 공사장 현황**
  - URL: https://www.data.go.kr/data/15028600/fileData.do
  - 공사명·공사기간·시공사·세대수

### 2차 공간 데이터 (정성)
- **카카오맵 스카이뷰 / 네이버맵 위성사진** — 현재 시공 중 크레인 위치 확인
- **V-World** — 지적도, 측정도구로 좌표 추출

### 3차 보조 데이터 (블로그·뉴스)
- 호갱노노, 정비사업 정보몽땅, 부동산 뉴스 — 시공사 정보·진행 상황

## 검증 절차 — 5 단계

### Step 1: 후보 사례 추출 (1일)
1. 공공데이터 CSV 다운로드 (마포구 착공신고)
2. 필터링:
   - 주용도: 도시형생활주택
   - 대지면적: 300 ~ 700 ㎡ (협소대지)
   - 최대지상층수: 5 ~ 12 층
   - 착공일: 2022 ~ 2024 (위성사진 확인 가능 시기)
3. 결과: 5~10건의 후보 케이스 추출 → 표 정리

### Step 2: 부지 정보 수집 (2일)
각 후보 케이스마다:
1. V-World 에서 부지 형상 측정 (정점 좌표)
2. 인접건물 정보:
   - V-World 건축물정보 → 층수, 높이
   - 카카오맵 거리뷰 → 시각적 확인
3. 도로 정보:
   - V-World 측정도구 → 도로폭
   - 점용가능 여부 (대로/골목)
4. 결과: 각 케이스의 JSON 부지 정의 파일

### Step 3: 우리 시스템 실행 (1일)
1. 각 JSON 부지 → 우리 시스템 적용
2. NSGA-II dual-branch 실행
3. Knee solution 추출:
   - 추천 위치 (x, y)
   - 추천 모델
   - 지브 길이
4. 결과: 사례별 추천 표

### Step 4: 실제 시공 비교 (2일)
1. 각 케이스에 대해 시공 중 시기의 위성사진 확인:
   - 카카오맵 스카이뷰 슬라이더 (과거시점)
   - 네이버맵 거리뷰 시점별
2. 실제 타워크레인 위치 좌표 추출 (위성사진에서 측정)
3. 크레인 모델 식별 (가능한 경우):
   - jib 길이 추정 (그림자/이미지에서)
   - 모델 유형 (러핑/T형)
4. 결과: 사례별 [우리 추천] vs [실제 시공] 비교 표

### Step 5: 통계 분석 + 보고서 작성 (1일)
1. 위치 오차: |우리 추천 좌표 - 실제 좌표| (m)
2. 모델 유형 일치율: (러핑/T형) %
3. 도로점용 여부 일치율: %
4. 통계 검정: Wilcoxon test (우리 추천이 random 추천보다 가까운가)
5. 결과: 정량 검증 표 + 정성 분석 + 한계 명시

## 예상 결과

### 가장 가능성 높은 결과
- **러핑 크레인 일치율 80% 이상** (실무 표준)
- **도로점용 여부 일치율 70% 이상**
- **위치 오차 평균 3~5m**

### 만약 일치율 낮으면?
- 그것도 의미 있음 → "우리 알고리즘이 더 엄격한 안전 제약을 적용하기 때문에 실무와 다를 수 있다" 라고 보고
- 어느 사례에서 어떤 제약 때문에 차이 나는지 정량 분석

## 한계 명시

1. **위성사진 시계열 제한**: 크레인 설치 정확 시점 못 잡을 수 있음
2. **크레인 모델 식별 한계**: 위성에서 보이는 건 jib 길이 정도, 정확 모델은 어려움
3. **실제 시공 = 최적 시공 아님**: 실무는 비용·관계사·관행 등 우리 모델 외 요인 영향
4. **표본 크기**: 5~10건은 통계적으로 작음 — pilot study
